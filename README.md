# UI4Bot — 具身智能人机交互界面参考库

本仓库收集了机器人、无人机、机器狗等具身智能体在**作业场景**中的人机交互（HMI）界面截图与录屏，供产品、设计与研发同事汲取灵感，思考更美观、更人性化、更科学、体验更好的控制台与监控大屏。

**覆盖载体**：地面机器人 / 机器狗、无人机（UAV）、物流巡检平台等。

---

| 目录 | 说明 |

## 一、界面的功能分区

具身智能控制台通常不是单一页面，而是**多模态、多层级**的信息面板组合。可参考以下三区划分布局与信息优先级。

### 1. 本体基本状态与人工控制

**目标**：让操作员在 3 秒内回答——「哪台设备、什么状态、我能否接管」。

| 要素 | 参考做法 | 设计要点 |
|------|----------|----------|
| 设备列表 / 编队 | 左侧栏列出 UAV_1、ROBOT-001 等，附在线 / 任务中 / 充电等状态 | 一机一行，状态用颜色 + 短文案，避免长段落 |
| 关键遥测 | 电量（百分比 + 电压）、位姿（X/Y/Z 或 x/y/yaw）、控制模式（手动 / 自动 / AI） | 仪表盘或环形进度条；异常值高亮，正常态克制用色 |
| 本体可视化 | 四足机器狗 3D 线框、无人机图标 | 帮助非技术用户建立「我在控哪台机器」的心智模型 |
| 人工接管 | 紧急停止（全局置顶、红色）、暂停 / 继续 / 取消、单点导航、键盘遥控、手自动切换 | **安全动作永远可达**；与 AI 自主模式边界清晰，防误触 |
| 全局态势 | 顶栏：系统就绪、执行中、在线数；模式切换：手动 / AI / 打断 | 宏观状态一行展示，不抢占地图与视频主视野 |

**参考界面**：`pic/image.png`（AERIALCLAW 无人机控制台）、`pic/企业微信截图_*.png`（机器狗巡检物流大屏）。

---

### 2. 视频流

**目标**：提供「机器所见」，支撑远程判情、巡检取证与操作信心。

| 要素 | 参考做法 | 设计要点 |
|------|----------|----------|
| 主视频 | 居中或大区域实时画面（仿真 / 实机 WebRTC） | 主摄占最大面积；叠加时间戳、设备 ID、码率等轻量 OSD |
| 多光谱 | 可见光 + 红外 / 热成像双路并列 | 巡检、安防场景常见；路数 >2 时用标签页或画中画 |
| 云台与视角 | PTZ 控制、2D/3D 地图联动视角 | 视频与地图点击联动（点哪看哪）降低认知负担 |
| 标定与诊断 | 角落输出标定日志、连接状态 | 技术信息可折叠，默认不干扰操作员 |

**参考界面**：机器狗大屏左侧「可见光视频」「红外光流」；无人机控制台中央仿真/实拍主画面。

---

### 3. 空间地图（语义地图）

**目标**：回答「机器在哪、要去哪、周围环境如何」，是具身 UI 的**空间锚点**。

| 要素 | 参考做法 | 设计要点 |
|------|----------|----------|
| 地图形态 | 2D 俯视语义图（建筑、区域标签）；3D 点云 / 体素地图（粉青配色区分表面语义） | 按场景选 2D（调度）或 3D（精细导航）；支持一键 2D/3D 切换 |
| 语义信息 | 地标名称（商圈、楼栋）、区域类型、楼层（1F/2F/全部） | 语义层可开关，避免点云满屏时丢失「这是哪」 |
| 机器人态势 | 当前位置、航向线、高度 h=、速度矢量、探测范围同心圆 | 机位与轨迹颜色与 UI 主色一致（如蓝=本机、绿=路径） |
| 地图工具 | 导航点、区域框选、虚拟墙、路径显示、视角跟随、重定位 | 工具条贴地图边缘；高频操作（跟随、重置）放大可点区域 |
| 模式切换 | 导航模式 / 建图模式 / 切图 / 建任务 / 任务列表 | 模式与权限绑定，建图等高危模式需二次确认 |

**参考界面**：`pic/image.png` 右侧 SITUATION MAP；机器狗大屏中央 3D 点云与底部地图操作栏。

## 二、机器人作业相关（作业全生命周期）

将 UI 按**作业前 → 作业中 → 作业后**组织，比按「页面菜单」更符合操作员心智，也便于做流程化引导与审计。

### 1. 作业前：自检（Pre-flight / Pre-task）

**目标**：未通过自检不允许多数自主任务启动，减少现场事故与无效出动。

| 检查项 | UI 表现建议 |
|--------|-------------|
| 姿态 / 位置 | 地图上的重定位状态、是否「已选重定位点」；位姿数值与「就绪」徽章 |
| 电量 | 环形仪表 + 阈值色（如 &lt;20% 禁止起飞/出站）；是否充电中 |
| 硬件 / 末端 | 各子系统 OK/WARN/ERR 列表；机械臂、夹爪、传感器在线状态 |
| 网络 / 链路 | 遥控链路、图传、云端延迟；丢包率简要指示 |
| 温度 / 环境 | 电机、电池、环境温度；超温阻断任务 |
| 综合就绪 | 顶栏「系统就绪」+ 自检清单可展开（逐项勾选动画） |

**参考**：无人机控制台「系统就绪」+ 左侧 UAV 电量与坐标；机器狗「系统运行状态」电压与控制状态。

---

### 2. 作业中：任务记录与硬件监控（In-task）

**目标**：操作员能**看见进度、理解 AI 在做什么、随时介入**，而不是只看一个「运行中」。

| 模块 | 参考做法 | 设计要点 |
|------|----------|----------|
| 任务进度 | 任务列表 + 进度条（如大厅巡逻 100%）；子步骤「第 4/4 点」 | 当前任务置顶；历史步骤可折叠 |
| 计划与执行 | PLAN 步骤列表 + SYSTEM 每步耗时与成功/失败图标 | 自主任务必备：**可解释性**；慢步骤自动标黄 |
| 自然语言态 | LLM OUTPUT 描述当前观察与决策（「已到达第二检查点…」） | 降低非研发人员理解自主行为的成本 |
| 硬件监控 | 实时电量、速度、告警区；视频 + 地图双通道交叉验证 | 告警分级：提示 / 警告 / 严重；严重告警可联动暂停 |
| 人工干预 | 打断、暂停、单点导航、虚拟墙 | 干预后 UI 明确显示「已切手动」及恢复自主入口 |
| 事件流 | 底部时间轴日志，绿勾成功、时间戳对齐 | 支持导出，便于事后对齐视频与传感器数据 |

**参考**：`pic/image.png` 底部 LLM / PLAN / SYSTEM 三栏 + 事件日志；机器狗右侧任务列表与底部任务执行状态。

---

### 3. 作业后：总结报告（Post-task）

**目标**：形成可归档、可复盘、可对接工单或客户的**结构化产出**，而非仅「任务完成」四个字。

| 内容 | UI 表现建议 |
|------|-------------|
| 执行摘要 | 任务名、耗时、里程、覆盖点位、完成率 |
| 异常与告警 | 作业期间 WARN/ERR 汇总；关联时间点跳转日志与录像 |
| 感知结果 | 巡检发现、目标列表（「已获目标」）、截图/片段索引 |
| AI 报告 | 独立 Tab「执行报告」；LLM 生成的自然语言总结 + 结构化字段 |
| 数据沉淀 | 主导航「数据管理」：历史任务、轨迹回放、导出 PDF/JSON |

**参考**：无人机控制台右侧「执行报告」Tab；机器狗「数据管理」模块与已完成任务状态。

## 三、跨场景设计原则（可复用的灵感清单）

1. **宏观 → 微观信息层次**：顶栏全局态 → 中央空间+视频 → 底部逐步日志；扫一眼知健康，下钻知原因。
2. **多模态互证**：同一时刻，地图位置、视频画面、遥测数字、AI 文字应能相互印证，减少「界面说正常但视频不对」的信任危机。
3. **颜色即语义**：绿=就绪/成功，蓝=本机/进行中，橙=警告，红=急停/错误，紫=AI/LLM；全项目统一，勿一屏十套色板。
4. **安全动作优先**：急停、暂停、切手动在视觉与交互层级上高于普通按钮（尺寸、颜色、固定位置）。
5. **AI 可解释**：自主模式下展示「计划步骤 + 自然语言说明 + 每步耗时」，避免黑盒焦虑。
6. **暗色控台 + 高对比数据**：长时间监控减轻眼疲劳；地图与曲线用有限强调色，避免大面积纯白。
7. **密度与留白平衡**：工业大屏信息密度高，靠面板边框、分组标题、可折叠区维持可读性；避免所有指标同等字号。
8. **流程化而非功能堆砌**：用「自检 → 执行 → 报告」串联现有模块，新同事也能按流程找到对应界面。

## 四、如何使用本仓库

1. **产品 / 设计**：从 `pic/` 挑选与己方载体（狗/无人机/臂）最接近的稿，按「三区 + 三阶段」做差距分析。
2. **研发**：对照「作业中」的 PLAN/SYSTEM/日志分层，设计前后端事件模型与 WebSocket 推送字段。
3. **评审**：用本文第四节原则做 Checklist，而非仅凭主观「好不好看」。

欢迎同事在借鉴时补充标注：场景名称、载体类型、亮点与待改进点，便于仓库持续演进。
